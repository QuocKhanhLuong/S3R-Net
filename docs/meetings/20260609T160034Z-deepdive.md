## External Signals (Researcher)

- Adaptive multi-teacher weighting is already an established KD direction, so
  the novelty cannot be "learned teacher weights" alone. The stronger claim is
  dense, student-aware, cardiac-specific reliability routing.
- Medical segmentation KD is moving toward heterogeneous foundation teachers and
  adaptive gates, which supports using CineMA and MedSAM2 together but raises the
  ablation bar.
- Boundary/body decoupling is a strong precedent. The proposed split should
  explicitly route CineMA semantic/ROI filling and MedSAM2 boundary/shape cues,
  then validate the split with DSC plus HD95/ASSD and boundary/surface metrics.
- Uncertainty-aware medical segmentation supports using entropy/margin maps, but
  student uncertainty must be detached and warm-started to avoid confirmation
  bias.
- Current analytic agreement saturation around `0.996` is a practical reason to
  move beyond `A=exp(-JS(P_MedSAM2,P_CineMA))` as the only reliability signal.

## Internal Health (Architect)

- The current implementation is analytic agreement fusion, not a trainable gate.
  `agreement_aware_fusion()` derives `W_M3/W_C/W_fuse` from detached teacher maps
  and returns detached diagnostics.
- Fit is good: KD is computed after student logits are available, so detached
  student entropy, margin, and boundary uncertainty can be added without changing
  teacher wrappers or cache schema.
- High implementation risk: a learned gate cannot live only inside a stateless
  loss helper. It must be an `nn.Module` whose parameters are registered with the
  optimizer, checkpoints, AMP/DDP, and resume logic.
- Keep teacher cache unchanged. The gate can consume existing `P_M3`, `C_M3`,
  `P_C`, `C_C`, `B_C`, teacher disagreement, and detached student signals.
- Tests must cover disabled fallback equivalence, shape/bounds, warmup behavior,
  detached inputs, nonzero gate gradients, optimizer inclusion, checkpoint
  resume, and disagreement behavior.

## Recommendations (PM)

1. **Frame the gate as a student-aware teacher-conflict router**
   Priority: high
   Effort: small

   The contribution should be a dense reliability gate that routes complementary
   image-space teacher knowledge where analytic teacher agreement is saturated.

2. **Implement it as a registered `nn.Module`**
   Priority: high
   Effort: medium

   `StudentAwareReliabilityGate` must be owned by the trainer/model path so its
   parameters enter optimizer state and checkpoints. Only teacher tensors and
   logging copies should be detached.

3. **Prove fallback equivalence first**
   Priority: high
   Effort: small

   Disabling the learned gate must reproduce the current analytic agreement
   behavior before any paper experiment is trusted.

4. **Prioritize a three-way main experiment**
   Priority: high
   Effort: medium

   Run: supervised S3R, current analytic dual-teacher KD, learned student-aware
   reliability gate. Keep cache, split, optimizer, schedule, and inference fixed.

5. **Ablate only signals that defend novelty**
   Priority: high
   Effort: medium-large

   Required ablations: teacher confidence only, teacher disagreement, entropy and
   margin, boundary/interior maps, detached student uncertainty, uncertainty
   warmup, `W_ignore` on/off, and role swap.

6. **Add reviewer-facing reliability analysis**
   Priority: medium
   Effort: medium

   Show gate maps, teacher-selection histograms, boundary-vs-interior behavior,
   high-disagreement pixel strata, and suppressed-teacher cases.

7. **Keep the module opt-in until evidence is strong**
   Priority: medium
   Effort: small

   Do not make the learned gate the default until it beats analytic fusion across
   overlap, surface, and reliability metrics.
