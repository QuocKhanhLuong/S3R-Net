## Internal Health (Architect/Dev)

- The previous dual-teacher path was analytic agreement fusion only.
- A learned gate must be an `nn.Module`, not a stateless loss helper, otherwise
  optimizer/checkpoint/resume handling will silently miss trainable parameters.
- The smallest safe implementation surface is the existing KD path:
  `compute_dual_teacher_kd_loss` receives an optional reliability gate and epoch,
  while the trainer owns gate construction, optimizer groups, checkpointing, and
  startup reporting.
- Gate params need a separate optimizer group and gradient clipping must cover
  all optimizer params, not only the S3R student model.
