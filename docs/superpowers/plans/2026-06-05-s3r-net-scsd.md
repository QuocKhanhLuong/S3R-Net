# S3R-Net And S3R-SCSD Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Promote the tested SSR_FULL prototype into the main S3R model family and add cache-based phased dual-teacher distillation.

**Architecture:** S3R keeps a feature path `F_t` and explicit spectral state `S_t`; each transition updates both via `S3RTransitionBlock`. Distillation is cache-first: supervised S3R training remains independent, and teacher outputs can be loaded from `.npz` files or generated from GT for smoke tests.

**Tech Stack:** Python, PyTorch, NumPy, SciPy when available, pytest, YAML configs.
